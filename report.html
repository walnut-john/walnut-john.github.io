Bayesian Data Analysis of Arrest Data from Pickens County, Ga¶

John Walker
Project - ISYE 6420 Bayes

Abstract¶

I performed a Bayesian data analysis on web-scraped arrest ("just busted") data from Pickens County, Ga. I begin with an exploratory data analysis of the data and several key variables. Then, I fit a linear regression model to estimate the effects of several variables on bond amounts charged. Lastly, I fit a Poisson linear regression model to estimate the effects of different types of seasonality on the number of people arrested per day.

Background and Introduction¶

For my Bayesian data analysis, I am analyzing arrest (or “just busted”) data from my hometown, Pickens County, Ga. I scraped the data from this website⁽¹⁾, and it includes fields such as the individual’s name, age, sex, race, etc. (all of which will be explored in a later section).

In this analysis, I seek to answer the following two questions using Bayesian methods:

  1. Is it possible to predict or explain the bond amounts charged given a set of input factors? In this section of the analysis, a linear regression model will be fit to the data with the bond amount as the response. I will investigate which variables are significant (based on their HPD) and whether or not the model offers significant predictive or explanatory power.
  2. Is it possible to predict the number of arrests per day by knowing the day of the week or the month of the year? In this section, I will fit a Poisson linear regression model to the daily number of arrests in order to see how much variation can be explained by these factors.

Data Description and EDA¶

The data was scraped from a just busted website⁽¹⁾, which is an aggregator of various US county's arrest data. At the time of writing, there were 3,383 records available for Pickens County, Ga, with the earliest record dating back to October 2018.

The data used for this analysis has been pre-processed/cleaned extensively. Pre-processing included aggregating bond amounts, putting all charges into a text string, using k-means to cluster the charges into similar buckets, computing charge per bond required, etc. Please see analysis.py in the code/src folder for the code that performs these transformations.

Bayesian Linear Regression for Bond Estimation¶

Model Description and Estimation¶

Below, I will construct the linear model for estimating the per charge bond amounts.

This model will be of the form:

$\hat{logbondcharge_i} = \beta_{intercept} + \beta_{age}*age + \beta_{gender}*gender + \beta_{race}*race + \beta_{chargecluster}*chargecluster$

This is a simplified equation. Note that several of the variables above are categorical, and that pymc3 will one-hot encode these variables to estimate a coefficient for each category of the variable.

Next, I will train the model.

Model Intepretation and Evaluation¶

We can see from the above that only a subset of the charge_cluster's, the intercept, and standard deviation are statistically significant.

My interpretation of this is that statistically, it doesn't matter so much as to what race, sex, or age the individual was when it comes to determining the bond amount. The charges themselves matter significantly more in determining the amount of bond that is charged to the offender. This finding makes intuitive sense, as the amount of bond a person is assessed should not have much to do with anything other than the crimes they were charged with.

However, it is likely that this model is not entirely accurate and should not be trusted fully. There were several under-represented subgroups in some of the variables (such as race and sex), which could lead to poor estimations of their coefficients. Additionally, the clustering of the charges into groups is not a perfect way to group similar charges together. This was done in this way for the sake of time and doesn't group things perfectly. Because of this noise and the underrepresented groups, the results must be taken with a grain of salt.

Directly below, I briefly evaluate the model output graphically via a histogram of the posterior predictive values vs. observed values of the response variable.

It can be seen clearly that while the model does produce an estimate that is generally in line with the observed values, it misses a significant amount of variation in the response. This could be due to missing independent variables, such as the income of the individual arrested.

Another potential issue affecting the model could be improper data transformations. Since I am trying to predict the average bond amount for each individual (total bond amount charged/number of charges), there could be a significant amount of variance in the group of charges for an individual. This should be accounted for by grouping all of an individual's charges into a cluster of similar individuals, but more than likely there are cases where this adds noise into the model.

I still think that a Bayesian multivariate model could work for this data if we had more observations for certain subgroups in the data, better data labelling for the charges, and more data points to feed into the model. However, in this current state, it's not a very accurate model as it misses much of the variation between bond amounts charged.

Below I will fit a Poisson linear regression model with the following form:

This choice of variables is to see whether there is any predictive or explanatory power in the seasonality of the time series, at several different levels. This model should indicate what time periods have a significant impact on the number of arrests per day.

Model Interpretation and Evaluation¶

It can be seen that the most significant variables are those for certain days of the week and the year 2019. The days of the week that are significant are Saturday, Sunday, Monday, and Thursday. The coefficients for these variables are all significantly negative (compared to the base case of Friday) at the 95% confidence level, indicating that arrests are lower for most days of the week compared to Friday. The year 2019 is also statistically significant at the 95% confidence level and is statistically positive compared to the base case of the year 2018. The fact that 2019 had a higher number of arrests than 2018 can be further evidenced by the monthly graph of the time series plotted in a section above.

It is interesting to see that after accounting for the daily and yearly seasonality, monthly seasonality shows no statistically significant effects across any of the months. This makes some sense, given that there doesn't appear to be any seasonality within the years in our data set, but is still somewhat surprising.

A caveat for this model's interpretation and estimation is that the dataset is very limited. We only have data back to October 2018 in this data set, and further complicating matters is that the COVID-19 pandemic did appear to have a significant effect on the number of arrests early in the pandemic (see March/April/May 2020 in the monthly time series chart). This alone could affect the estimation of the coefficients and therefore their interpretation.

Conclusions¶

I used Bayesian methods available in the pymc3 python library to fit linear models to a unique dataset of arrests.

In predicting bond amounts for an individual using linear regression, I found that while the shape of the data can be approximately with a linear model, much of the variation in the response variable is unexplained. I would be interested in exploring this topic further, possibly using Bayesian methods, other machine learning models, or just improving the dataset itself.

In explaining the seasonality of the time series of arrest counts per day, I found that a Bayesian fitted linear regression model can approximate the distribution of the data fairly well. The model also provides an explanation of which days of the week are more likely to see a higher number of arrests (via the coefficient values).

In using the pymc3 library, I enjoyed how easy it was to actually fit the model by using the simplified glm(), where you can input the model form in an "R-type" syntax. However, the time it takes to fit the model is very long compared to how long it takes to fit the same model in R using classical statistical techniques. Also, generating predictions using the models in pymc3 is (unless you have a simple model with a few parameters) seemingly not feasible. If I were fitting these models in practice, I would probably stick with R. However, if I needed to add informative priors on some of the parameters, and the model was simple enough in form, I would consider pymc3.
